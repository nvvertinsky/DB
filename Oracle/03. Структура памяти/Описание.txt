Три основные структуры памяти:

  - Глобальная область памяти (System Global Area - SGA)
    - Это область памяти, к которому могут обращаться практически все процессы Oracle. 
	
  - Глобальная область процесса (Process Global Area - PGA)
    - Это закрытая область, которая находится в отдельном каждом процессе. 
	
  - Глобальая область пользователя (User Global Area - UGA)
    - Область памяти связанная с конкретным сеансом
	- Если подключение к БД выполнено через разделяемый сервер, то память распологается в SGA. Таким образом памятью можно пользоваться любой из разделяемых процессов, посколько любой из них может считывать и записывать данные сеанса.
	- Иначе в PGA.
	
	
Режимы управление памятью.:
  - Автоматическое управление памятью (AMM).
    - Применяется к PGA и SGA.
	- Доступно только с 11g.
	- Нужно установить всего один параметр MEMORY_TARGET.
	- Oracle сам распределит память SGA, PGA как ему надо.
  
  - Автоматическое управление памятью SGA (ASMM).
    - Нужно установить размер параметра SGA_TARGET
  
  - Автоматическое управление памятью PGA.
    - Нужно установить размер параметра PGA_AGGREGATE_TARGET
	- Какой режим выбран контролирует параметр WORKAREA_SIZE_POLICY
  
  - Ручное управление SGA.
    - Нужно вручную выставлять параметры DB_CACHE_SIZE, SHARED_POOL_SIZE и т.д.
  
  - Ручное управление PGA
    - Нужно вручную выставлять параметры SORT_AREA_SIZE, HASH_AREA_SIZE
	- Какой режим выбран контролирует параметр WORKAREA_SIZE_POLICY
