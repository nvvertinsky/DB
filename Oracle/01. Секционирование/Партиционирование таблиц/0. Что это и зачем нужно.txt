Определение:
  Секционирование/партиционирование - это возможность разбить таблицы/индексы на несколько меньших, проще управляемых частей.



Зачем оно нужно?:
  1. Повышение доступности данных.
	- Если одна секция не доступна, то если мы не обращаемся к этой секции, то ошибки не будет.
	- Сокращение времени простоя. Например таблица в 100 гб, разбитая на 50 секций по 2гб, то вы можете восстанавливать ее после ошибок гораздо быстрее. Если повреждено секция 2 гб, то восстанавливать нужно 2гб, а не всю таблицу в 100 гб.
	
  2. Повышение производительности в операциях
	- Возможность распараллеливание DML и чтения данных. Лучше проявляется в хранилищах данных, чем в OLTP
	- Допустим у нас хранилище.
		- В хранилище крупная таблица и мы делаем запрос за один квартал.
		- Индексация квартала плохая идея, потому что такой индекс указывал бы на сотни тысяц строк. 
		- В результате будет полное сканирование таблицы, т.к. оно будет быстрее чем индекс. Получается придется сканировать всю огромную таблицу.
		- А если будет сделано секционирование по кварталу, то будет прочината только одна секция.
		- Так же в хранилищах данных применяются параллельные запросы. Параллельное сканирование диапазона индеса или параллельное быстрое сканирование индекса.
		
  3. Упрощение администрирования
	- Архивация фрагментов таблиц.
	- Перестройка не всего индекса, а только его части. Одну часть за другой. Для этого понадобится меньше ресурсов. А если будет откат из-за ошибки, то откатиться не вся работа, а только часть.
	- При налиичии таблицы в 100 гб и она разбита на 25 секций по 4 гб, то можно перестаивать секции одну за другой. Можно это делать еще в парралельных сеансах, чтобы быстрее сделать.
	
  4. Сокращение конкуренкции в системе OLTP:
    - Если есть объект испытывающий высокую конкуренцию, то его превращение в набор из нескольких объектов (секционирование) может дать эффект в виде сокращения такой конкуренции.
 
 
 
Виды секционирования: 
  1. Одноуровневое - range, list, hash
  2. Составное - два уровня, комбинация из range, list, hash
  3. Ссылочное секционирование
  4. Системное секционирование
  5. Гибридное секционирование - часть секции в БД, а часть на внешнем источнике
  
  

Когда что применяется:
1. range - когда есть данные, которые логически разделены по какому-то значению. (квартал, месяц, год).

2. list - когда есть столбец, который представляет из себя какой-то список. Например список кодов штатов, региона и другие атрибуты выраженные с помощью кода

3. hash - когда данные не имеют естественных диапазонов значений.
