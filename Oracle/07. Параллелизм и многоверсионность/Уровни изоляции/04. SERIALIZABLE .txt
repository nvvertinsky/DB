Характеристики:
  - Любая прочитанная строка гарантировано будет такой же при повторном чтении, например:
select * from t;
begin dbms_lock.sleep(60*60*24); end;
select * from t;

  - Изменения сделанные другими транзакциями для запросов выше остаются невидимыми
  - В Oracle реализовано так, что согласованность чтения оператора, расширяется до уровня транзакции


Как работает: 
  - Все так же применяются сегменты отката. Только теперь на момент начала транзакции.


Возможные ошибки:
  - ORA-08177: Cant serialize access for this transaction. Будем получать при попытке обновления строки, которая изменилась с момента начала транзакции.


Когда применять: 
  - Высока вероятность, что никто не модифицирует те же самые данные
  - Требуется согласованность чтения на уровне транзакции
  - Транзакции будут короткими. Что поможет с первым условием. 

Команда:
alter session set transaction serializable;
